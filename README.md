# FUZZ Scanner V2.1 — Burp Suite 漏洞检测插件
一个用于 Burp Suite 的双模式 Fuzz 扫描插件，支持 SSTI（服务端模板注入）、SQL 注入等漏洞的自动化检测，具备深度嵌套参数解析能力。
---

## 功能特性

### 双扫描模式

| 模式                      | 说明                                                       | 适用场景                                     |
| ------------------------- | ---------------------------------------------------------- | -------------------------------------------- |
| **模式1：替换整个请求体** | 清空原始请求体，替换为自定义 PoC 。GET 请求自动转为 POST。 | 对请求体结构无依赖的检测（如 SSTI 盲注、fastjson反序列化）     |
| **模式2：替换参数值**     | 保持原始请求结构，逐个替换每个参数的值为自定义PoC。         | 依赖原有参数上下文的检测（如 SQL 注入、文件读取、系统命令注入等） |

### 模式2 高级选项

| 选项                               | 说明                                                         |
| ---------------------------------- | ------------------------------------------------------------ |
| **注入 Cookie 参数**               | 是否对 Cookie 中的参数进行注入测试                           |
| **注入 Header 参数**               | 是否对自定义 Header 中的参数进行注入测试                     |
| **使用完整 POC 字符串**            | 开启：参数值替换为完整 PoC；关闭：仅用提取的 payload 值 (仅针对特定poc，比如完整poc结构为fragment=`__${7*7}__` 的时候，关闭此选项仅会提取__${7*7}__放入到参数)     |
| **追加 POC 参数到请求**            | 在保留原参数的基础上，追加 PoC 的 key=value 结构 (与mode 1功能略有区别，仅针对仅针对特定poc，比如完整poc结构为fragment=__${7*7}__ 、原始请求假设为key=value的时候，开启此选项，请求后的参数为key=value&fragment=__${7*7}__)             |
| **保留原始值再拼接 Payload**       | 开启：`username=admin` → `username=admin' or 1='1`；关闭：`username=admin` → `username=' or 1='1` ,此功能针对依赖原有参数场景|
| **URL 编码替换值**                 | 是否对替换的 payload 值进行 URL 编码 (仅对特殊符号、空格进行URL编码)                        |

### 深度参数解析

支持自动识别并提取以下格式中的嵌套参数：

- **application/x-www-form-urlencoded** — 键值对，自动解析值中嵌套的 JSON
- **application/json** — 递归提取所有叶子节点，完美处理重复 Key
- **multipart/form-data** — 提取表单字段，自动跳过文件字段
- **text/xml / application/xml** — 提取叶子标签的文本内容
- **未知类型** — 智能尝试 JSON / Form 解析

### 其他特性

- **白名单域名过滤** — 正则匹配，跳过不需要检测的域名
- **静态文件过滤** — 自动跳过 jpg/png/css/js 等静态资源
- **监控范围选择** — 可独立开关 Proxy / Repeater 流量监控
- **右键菜单** — 支持 "Send to FUZZ Scanner" 手动发送
- **请求/响应查看器** — 点击结果即可查看原始请求和响应
- **URL 去重** — 同一 URL 只扫描一次

## 安装方法

1. 在 Burp Suite 中打开 **Extender** 标签页
2. 点击 **Add** → 选择 **Python** 作为扩展类型
3. 选择 `ssti_scanner.py` 文件
4. 确保已配置 Jython（Burp Suite → Extender → Options → Python Environment）

> **依赖**：Burp Suite Professional / Community + Jython 2.7+

## PoC 配置

### 配置文件位置

```
~/fuzz_scanner.txt
```

（即用户主目录下的 `fuzz_scanner.txt` 文件）

### 配置格式

每行一条规则，格式为：

```
POC字符串|||匹配特征字符串
```

### 示例

```
fragment=__${7*7}__|||49
fragment=__{{7*7}}__|||49
fragment=__<%=7*7%>__|||49
```

- `|||` 左侧为发送的 PoC payload
- `|||` 右侧为响应中需要匹配的特征字符串（如计算结果 `49`）
- 如果响应中包含匹配特征，则标记为 **Found!**（存在漏洞）

### 内置默认 PoC

插件内置了 4 条针对 Java 模板引擎（Freemarker/Velocity）的 SSTI 检测 PoC，即使没有 `fuzz_scanner.txt` 文件也能正常使用。你也可以根据自己的需求编写任意类型的 PoC（SQL 注入、XSS、命令注入等）。

## 使用说明

### 界面布局

```
┌─────────────────────────────────────┬──────────────────┐
│  # │ Time │ URL        │ Method│状态 │  POC列表编辑器   │
│─────────────────────────────────────│                  │
│  详细结果表格（POC / 匹配 / 结果）  │  POC文件路径     │
│─────────────────────────────────────│  保存POC 按钮    │
│      请求查看器  │  响应查看器      │                  │
└─────────────────────────────────────┴──────────────────┘
```

### 操作步骤

1. 在 **Proxy** 或 **Repeater** 中正常浏览/发送请求
2. 插件自动捕获流量并开始扫描（需勾选「启用插件」）
3. 点击左侧表格中的 URL，查看右侧的详细测试结果
4. 点击详细结果中的某条记录，下方会显示对应的请求和响应
5. 标记为 **Found!** 的结果即为可能存在漏洞的请求

### 右键菜单

在 Burp 任意位置（Proxy/Repeater/Intruder 等）右键 → **Send to FUZZ Scanner** 即可手动触发扫描。

## 注意事项

- 插件默认**不会自动启用**，需勾选「启用插件」复选框
- 白名单域名默认过滤 Google 和 Baidu，可按需修改或关闭
- 模式2 会产生较多请求（参数数量 × PoC 数量），建议在授权测试环境中使用
- PoC 配置文件使用 UTF-8 编码，修改后需点击「保存POC」按钮生效
